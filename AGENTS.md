## Unit Template

The canonical blank unit template lives at `docs/ngss-unit-template.md`.
When the user asks to create a new unit (e.g. "create a unit for standards X, Y, Z" or "draft a unit on phenomenon X"), read this file first and use it as the structural guide. It defines the exact section headings, field labels, and formatting that the markdown parser expects.

Key things the template enforces:
- Summary Table field labels must be exactly: `Activity / Big Idea`, `What we learned`, `How it helps my understanding`, `What do I need to modify in my model`
- Loop resources use the format: `[Title (type)](URL)` where type is one of: article, youtube, google-doc, simulation, other
- The Quick Reference planning table is auto-generated by the app — do NOT include it in the output
